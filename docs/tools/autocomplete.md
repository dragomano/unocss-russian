# Автодополнение {#autocomplete}

Утилиты автодополнения для UnoCSS: `@unocss/autocomplete`. Они встроены в <a href="/play" target="_blank" rel="noreferrer">песочницу</a> и [расширение VS Code](/integrations/vscode).

## Использование {#usage}

### Статические правила {#static-rules}

Статические правила, подобные этим, будут работать без какой-либо конфигурации.

```ts
rules: [
  ['flex', { display: 'flex' }]
]
```

### Динамические правила {#dynamic-rules}

Для динамических правил вы можете предоставить дополнительный объект `meta` в правиле и указать шаблон автодополнения:

```ts
rules: [
  [
    /^m-(\d)$/,
    ([, d]) => ({ margin: `${d / 4}rem` }),
    { autocomplete: 'm-<num>' }, // <-- этот
  ],
]
```

Шаблон использует простой DSL для указания предложений автодополнения. Синтаксис:

- `(...|...)`: логические группы «ИЛИ», использующие `|` в качестве разделителя. Будут использоваться в качестве предложений, когда совпадают некоторые из групп.
- `<...>`: встроенные сокращения. В настоящее время поддерживаются `<num>`, `<percent>` и `<directions>`.
- `$...`: вывод из темы. Например, `$colors` перечислит все свойства объекта `colors` темы.

## Примеры {#examples}

### Пример 1 {#example-1}

- **Шаблон**: `(border|b)-(solid|dashed|dotted|double|hidden|none)`
- **Ввод**: `b-do`
- **Предложения**: `b-dotted`, `b-double`

### Пример 2 {#example-2}

- **Шаблон**: `m-<num>`
- **Ввод**: `m-`
- **Предложения**: `m-1`, `m-2`, `m-3`…

### Пример 3 {#example-3}

- **Шаблон**: `text-$colors`
- **Ввод**: `text-r`
- **Предложения**: `text-red`, `text-rose`…

### Пример 4 {#example-4}

Для нескольких шаблонов:

- **Шаблон**: `['(border|b)-<num>', '(border|b)-<directions>-<num>']`
- **Ввод**: `b-`
- **Предложения**: `b-x`, `b-y`, `b-1`, `b-2`…
- **Ввод**: `b-x-`
- **Предложения**: `b-x-1`, `b-x-2`…
